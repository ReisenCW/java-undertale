# 2025-2026学年第一学期 / 420407 - 软件设计模式 / 期末项目
## 要求
每组学员需利用设计模式对现有软件项目的源代码进行重构，**强烈建议**重构团队成员之前开发的项目，特殊情况下也可考虑开源项目。

团队需应用课堂讲授的**五种设计模式**（含一种创建型模式、两种结构型模式、两种行为型模式），以及**一种课堂未覆盖的额外设计模式**，可参考维基百科设计模式列表（ https://en.wikipedia.org/wiki/Design_Patterns ）探索额外模式。

重构过程中，团队需探索并实践AI工具的多种用途，包括但不限于识别重构机会、提出设计方案、应用设计模式转换源代码、评估重构质量。

团队需提交重构后的源代码和项目报告，并最终完成项目展示。

## 提交要求
每组需提交包含以下材料的压缩包：
a) 完整的重构后源代码：若总大小超过20MB，可仅提交重构部分；重构后的源代码需通过注释明确标注，例如“使用XXX模式重构”。
b) 一份项目报告，按以下顺序包含相关内容：
1) 软件项目名称、团队编号、所有成员的全名及学号、联系电话、电子邮箱及其他相关信息；
2) 软件项目简要说明（如项目背景、主要功能）；
3) 重构工作详细阐述：重点说明变更的源代码文件、重构前后的主要差异（附UML类图示意），并解释变更原因（如解决的问题、获得的收益等）；
4) 重构过程中AI工具的使用详情，及对最佳实践、局限性、挑战与解决方案、经验教训的反思讨论；
5) 其他相关或有讨论价值的补充信息。
**注意**：报告需用英文撰写，提交格式为PDF。
c) 最终展示幻灯片：第一页需包含软件项目名称、团队编号、所有成员的全名及学号。
**注意**：幻灯片需用英文制作，提交格式为PDF。

请将源代码（a）与文档（b、c）分别存入不同文件夹，压缩为一个ZIP文件后通过Canvas提交。

## 展示要求
每组需在10分钟内完成项目展示，随后进行问答环节，展示需用英文进行。团队需完成以下内容：
a) 简要介绍项目及重构动机；
b) 明确展示并解释六种设计模式的应用：需通过带清晰说明的UML类图，详细对比重构前后的源代码，阐述变更的合理性、必要性及带来的收益；
c) 说明重构过程中AI工具的使用情况，及对最佳实践、局限性、挑战与解决方案、经验教训的反思讨论；
d) 讨论重构过程中涉及的其他重要问题及对应解决方案。

## 评分标准
| 评分指标                             | 权重 |
| ------------------------------------ | ---- |
| 一种创建型模式的应用（含详细说明）   | 10%  |
| 两种结构型模式的应用（含详细说明）   | 20%  |
| 两种行为型模式的应用（含详细说明）   | 20%  |
| 一种额外设计模式的应用（含详细说明） | 15%  |
| AI工具的使用（含详细讨论）           | 15%  |
| 项目报告质量                         | 10%  |
| 展示质量                             | 10%  |
